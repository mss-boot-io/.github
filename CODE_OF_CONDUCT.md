# Code of Conduct

## Our pledge

We want `mss-boot-io` to be a practical, respectful, and useful open source
community. Everyone is expected to communicate with patience, technical honesty,
and care for maintainability.

## Expected behavior

- Be respectful when discussing trade-offs, defects, and design decisions.
- Assume contributors are trying to help, then ask for details when context is
  missing.
- Keep issue and pull request discussions focused on reproducible facts,
  project goals, and user impact.
- Disclose security concerns through the security policy instead of public
  issues.

## Unacceptable behavior

- Harassment, threats, insults, or repeated personal attacks.
- Publishing private information without consent.
- Deliberately disruptive comments, spam, or off-topic promotion.
- Pressuring maintainers to bypass review, testing, or release rules.

## Enforcement

Maintainers may hide comments, close issues, block users, or limit participation
when behavior harms the project or community. Enforcement decisions should be
documented privately by maintainers when possible.
