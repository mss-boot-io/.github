# Security Policy

## Reporting a vulnerability

Please do not open a public issue for suspected vulnerabilities.

Use GitHub Security Advisories for the affected repository when private
vulnerability reporting is enabled. If that setting is not available yet, contact
the maintainer through the private channel listed in the repository-specific
security policy.

The organization still needs a final public security contact. Until then,
repository maintainers should treat private GitHub advisories as the preferred
intake path.

## Response expectations

- Acknowledge valid private reports within 7 days when possible.
- Triage severity, affected versions, and exploitability before public
  disclosure.
- Prepare a fix, release note, and upgrade guidance before disclosing details.
- Credit reporters when they request it and when disclosure is safe.

## Supported versions

The active `main` branch and the latest tagged release are supported by default.
Older versions are handled case by case until the project publishes a formal
long-term support matrix.
