# mss-boot-io

`mss-boot-io` builds a governance-first, agent-readable Go microservice admin
framework for maintainable production systems.

The organization is moving from "rapid scaffolding" toward an AI-era open source
operating model: clear contracts, reproducible environments, automated quality
gates, security-first releases, and documentation that can be read by both
people and AI agents.

## Start here

- Documentation: [docs.mss-boot-io.top](https://docs.mss-boot-io.top)
- Admin preview: [admin-beta.mss-boot-io.top](https://admin-beta.mss-boot-io.top)
- API preview: [admin-api-beta.mss-boot-io.top](https://admin-api-beta.mss-boot-io.top)

## Core projects

| Repository | Role |
| --- | --- |
| [mss-boot](https://github.com/mss-boot-io/mss-boot) | Go framework foundation. |
| [mss-boot-admin](https://github.com/mss-boot-io/mss-boot-admin) | Backend API, RBAC, system configuration, notifications, and operational services. |
| [mss-boot-admin-antd](https://github.com/mss-boot-io/mss-boot-admin-antd) | React/Ant Design admin console deployed through Cloudflare. |
| [mss-boot-docs](https://github.com/mss-boot-io/mss-boot-docs) | Public documentation, delivery workflow, AI memory, and governance records. |

## Contribution model

Issues and pull requests are expected to include reproduction context, test
evidence, security impact, documentation impact, and release impact. Automation
will help triage and review metadata, while maintainers keep final decisions
human-owned.

Security reports should not be opened as public issues. Use the repository
security policy or GitHub Security Advisories when enabled.
